[멀티스레드 문제] 스마트폰에서 멀티태스킹 구현하기.
🎵 문제
하나의 프로그램(스마트폰)에서 두 가지 작업(음악 재생, 웹서핑)이 동시에 실행되는 멀티태스킹 환경을 자바 스레드를 이용해 구현합니다.
✅ 요구사항
독립적인 작업: '음악 재생'과 '웹서핑' 기능은 서로 영향을 주지 않고 독립적으로 ON/OFF가 가능해야 합니다.
주기적인 출력: 각 기능이 실행 중일 때는 1초마다 정해진 메시지("음악 재생 중...", "웹서핑 중...")를 출력합니다.
사용자 제어: 사용자는 간단한 숫자 입력(1, 2, 0)을 통해 각 기능의 실행/종료 및 프로그램 전체 종료를 제어할 수 있어야 합니다.
💻 실행 예
1. 뮤직 ON/OFF | 2. 웹서핑 ON/OFF | 0. 종료
>> 1
음악 시작!
음악 재생 중...
>> 2
웹서핑 시작!
웹서핑 중...
음악 재생 중...
>> 1
음악 종료!
웹서핑 중...
>> 0
스마트폰 종료...

[새로운 문제] 코인노래방 타이머 만들기
🎤 문제
코인노래방 기계를 시뮬레이션하는 프로그램을 만듭니다. 기본으로 주어진 시간 동안 노래가 재생되고, 시간이 모두 지나면 자동으로 종료되는 타이머 기능을 자바 스레드로 구현합니다.

✅ 요구사항
기본 시간: 프로그램이 시작되면 기본 사용 시간으로 60초가 주어집니다.
노래 시작: 사용자가 1을 입력하면 노래가 시작되고, "노래를 시작합니다!" 메시지와 함께 남은 시간 카운트다운이 시작됩니다.
시간 표시: 노래가 진행되는 동안 10초마다 남은 시간을 화면에 출력합니다. (예: "남은 시간: 50초")
자동 종료: 남은 시간이 0초가 되면 "노래가 종료되었습니다." 메시지를 출력하고, 노래 시작 전 상태로 돌아갑니다.
프로그램 종료: 사용자가 0을 입력하면 프로그램을 완전히 종료합니다.

💻 실행 예
[코인노래방] 1. 노래 시작 | 0. 종료
>> 1
노래를 시작합니다!
남은 시간: 50초
남은 시간: 40초
남은 시간: 30초
남은 시간: 20초
남은 시간: 10초
노래가 종료되었습니다.

[코인노래방] 1. 노래 시작 | 0. 종료
>> 0
프로그램을 종료합니다.

[새로운 문제] 5초 타자 게임 만들기
⌨️ 문제
제한 시간 안에 화면에 나타난 단어를 정확하게 입력해야 하는 타이머 기반의 타자 게임을 자바 스레드를 이용해 구현합니다. 메인 스레드는 사용자 입력을 받고, 다른 스레드는 시간제한을 담당합니다.

✅ 요구사항
단어 목록: 프로그램은 "Java", "Thread", "Exception", "Keyboard" 등 여러 개의 영단어를 목록으로 가지고 있어야 합니다.

게임 시작: 사용자가 1을 입력하면 게임이 시작되고, 단어 목록에서 무작위로 하나의 단어가 화면에 제시됩니다.

시간제한: 단어가 제시된 순간부터 5초의 입력 시간이 주어집니다.

결과 판정:

성공: 5초 안에 제시된 단어와 정확히 일치하는 단어를 입력하면 "성공!" 메시지를 출력합니다.

오타: 5초 안에 단어를 입력했지만, 제시된 단어와 일치하지 않으면 "오타!" 메시지를 출력합니다.

실패: 5초가 지날 때까지 아무런 입력이 없으면 "시간 초과! (실패)" 메시지를 출력합니다.

프로그램 종료: 사용자가 0을 입력하면 프로그램을 종료합니다.
